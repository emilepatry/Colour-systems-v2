# Behavior Mapping (BMAP) Framework
*Purpose: Behavior change formula (B = M × A × P) and intervention toolkit for product design.*

A framework for designing effective behavior change interventions.

---

## What is the BMAP Formula?

The BMAP formula explains when and why behavior occurs:

```
Behavior = Motivation × Ability × Prompt
```

**All three elements must be present** for behavior to happen. If any one is missing or too low, the behavior won't occur.

---

## The Three Components

### Motivation (M)

What drives the desire to act. Motivation is emotional, not rational.

**Motivation drivers:**
- **Hope/Anticipation** — Expectation of positive outcomes
- **Fear** — Avoidance of negative outcomes  
- **Acceptance/Belonging** — Social connection and approval
- **Sensation** — Immediate pleasure or relief

**Key insight:** You cannot create motivation—you can only amplify existing motivation or connect to deeper motivations the user already has.

### Ability (A)

How easy or hard it is to perform the behavior. Ability has multiple dimensions:

| Dimension | Question |
|-----------|----------|
| **Time** | How long does it take? |
| **Money** | How much does it cost? |
| **Physical effort** | How much physical work? |
| **Mental effort** | How much thinking required? |
| **Social deviance** | Does this go against norms? |
| **Routine disruption** | Does this break existing habits? |

**Key insight:** It's usually easier to increase Ability than Motivation. Simplify the action before trying to boost motivation.

### Prompt (P)

The trigger that initiates the behavior at the right moment.

**Prompt types:**
- **Explicit prompts** — Notifications, reminders, CTAs
- **Implicit prompts** — Environmental cues, habit stacking
- **Social prompts** — Peer behavior, recommendations

**Key insight:** Without a well-timed prompt, even highly motivated users with high ability won't act.

---

## How to Diagnose Behavior Problems

When a desired behavior isn't happening, diagnose which element is the limiting factor:

| Symptom | Likely Cause | Intervention |
|---------|--------------|--------------|
| "They're not interested" | Low Motivation | Build hope, show value, connect to existing desires |
| "They just don't get it" | Low Ability (mental) | Simplify, provide guidance, use familiar patterns |
| "It's too much work" | Low Ability (effort) | Break into micro-actions, reduce steps |
| "They forget to..." | Missing Prompt | Add timely triggers, create environmental cues |
| "They don't see the value" | Low Motivation (anticipation) | Show outcomes, use social proof |

---

## BMAP Scoring Method

For each behavior you want to encourage:

1. **Define the behavior** specifically
2. **Score each element** on a 1-10 scale:
   - Motivation: How much do they want this?
   - Ability: How easy is this to do?
   - Prompt: How clear is the trigger?
3. **Identify the lowest score** — this is your limiting factor
4. **Design interventions** targeting that element

### BMAP Scoring Template

```
Behavior: [Specific action you want users to take]

Motivation Score: __/10
- What's driving them?
- What's holding them back?

Ability Score: __/10
- How much time/effort does it take?
- What skills or resources are needed?

Prompt Score: __/10
- When do they encounter the trigger?
- Is the trigger well-timed and clear?

Primary Gap: [M / A / P]
Intervention: [Specific lever to improve]
```

## Intervention Toolkit

### Motivation Boosters

| Intervention | How it works |
|--------------|--------------|
| Show progress indicators | Creates sense of advancement |
| Highlight peer success | Social proof builds hope |
| Create anticipation loops | Build excitement for outcomes |
| Reduce fear through guarantees | Lower perceived risk |
| Connect to identity | "People like you do this" |

### Ability Enhancers

| Intervention | How it works |
|--------------|--------------|
| Simplify first action | Reduce initial barrier |
| Provide templates/examples | Reduce mental effort |
| Reduce decision fatigue | Fewer choices = easier action |
| Offer practice mode | Build competence without stakes |
| Set smart defaults | No action required |

### Prompt Optimizers

| Intervention | How it works |
|--------------|--------------|
| Context-aware triggers | Right message at right time |
| Habit stacking | Attach to existing routines |
| Environmental cues | Physical reminders in context |
| Social prompts | Peer recommendations |
| Completion prompts | After one action, prompt the next |

---

## Design Principle: Surprise Patients with Value

Anticipate needs and offer timely support before it's requested. This means:
- Prompting at high-motivation moments (not random times)
- Reducing Ability barriers before users encounter them
- Building Motivation through visible progress and early wins

---

## Connecting BMAP to 6P Stories

Every panel transition in a 6P Story is a behavior that needs to occur. Use BMAP to analyze each transition:

| Panel Transition | BMAP Analysis | Intervention |
|------------------|---------------|--------------|
| Panel 2→3: Customer starts searching | Low M (doesn't believe solution exists) + High A + Missing P | Create hope-building content + timely trigger |
| Panel 3→4: Customer tries first solution | High M + Low A (too complex) + Present P | Simplify the action |
| Panel 4→5: Customer persists despite obstacle | Dropping M + High A + Present P | Add reassurance and progress signals |

## Common Patterns

### Habit Formation
```
Trigger (P) → Routine (A) → Reward (M) → Loop
```
For new habits: Start with tiny behaviors (high A), attach to existing triggers (reliable P), and celebrate small wins (boost M).

### Onboarding Behavior
Focus on **first action** success:
- Motivation is highest at signup
- Ability should be minimized (one simple action)
- Prompt should be immediate and clear

### Re-engagement Behavior
Focus on **prompt timing**:
- Motivation exists but is latent
- Ability hasn't changed
- Prompt is missing or poorly timed

## Quick Reference Checklist

### Before Designing an Intervention

- [ ] Defined the specific behavior I want?
- [ ] Scored Motivation, Ability, Prompt (1-10)?
- [ ] Identified the limiting factor?
- [ ] Matched intervention to the right element?

### When Motivation is the Limiting Factor

- [ ] Connected to existing user desires?
- [ ] Shown clear value/benefit?
- [ ] Used social proof?
- [ ] Reduced perceived risk?

### When Ability is the Limiting Factor

- [ ] Simplified the action?
- [ ] Reduced number of steps?
- [ ] Provided templates or examples?
- [ ] Eliminated decision points?

### When Prompt is the Limiting Factor

- [ ] Timed the prompt to high-motivation moments?
- [ ] Made the trigger clear and visible?
- [ ] Connected to existing routines?
- [ ] Tested different prompt formats?

---

## Red Flags: Common BMAP Mistakes

1. **Over-Prompting** — Adding notifications when the real issue is Ability or Motivation
2. **Motivation-Only Focus** — Assuming inspiration alone drives behavior
3. **Ignoring Context** — Not considering where/when behaviors occur in real life
4. **Feature Overload** — Adding complexity when simplification is needed
5. **Misaligned Interventions** — Solving for the wrong element
